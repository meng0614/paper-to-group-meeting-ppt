# Presentation Design Philosophy

Use this reference when a user provides a beautiful reference PPT. Learn its design philosophy, not its colors or fonts.

## Non-Negotiable Principles

1. One Slide One Message
   - Each slide expresses one core claim.
   - If a slide needs two messages, split it.

2. Visual First
   - Every non-cover slide has a visual center.
   - The visual center should occupy at least 40% of the slide.
   - The visual can be a paper figure, simplified diagram, comparison, flow, chart, or table.

3. Visual Hierarchy
   - Title > visual > explanatory text.
   - Do not give every element equal weight.

4. Whitespace First
   - Whitespace is a design element.
   - Do not fill the page simply because there is empty space.

5. Story First
   - Do not follow the paper section order mechanically.
   - Build the talk as:

```text
Problem -> Challenge -> Idea -> Method -> Result -> Takeaway
```

6. Audience First
   - Before each slide, answer:
     "If the audience sees this slide for only 5 seconds, what should they remember?"

7. Layout Quality
   - Prefer more slides over crowded slides.
   - A clear 15-slide talk is better than a cramped 10-slide deck.

## Reference PPT Policy

When a reference PPT is provided:

- Learn layout rhythm, whitespace, visual hierarchy, page density, and story flow.
- Do not copy the reference PPT's colors.
- Do not copy the reference PPT's fonts.
- Do not treat the reference PPT as a theme template.
- Treat it as a design philosophy sample.

## Lessons From Strong Technical Lecture Decks

When a user provides a polished technical PPT, extract these transferable patterns:

1. Stable master frame
   - Use consistent slide chrome such as a title zone, section label, page marker, subtle rail, or divider.
   - The exact colors and fonts must come from the current project style, not the reference deck.

2. Clear title hierarchy
   - The title should be the strongest text object on the slide.
   - Prefer assertion headlines that state the message, not generic section names.

3. Visual-text zoning
   - Separate the visual subject from explanation text.
   - Do not let figure captions, notes, or body text compete with the main visual.

4. Disciplined whitespace
   - Leave stable margins around the slide.
   - Empty space is part of the design; do not fill it with minor details.

5. Limited color roles
   - Use colors by role: primary text, structural accent, highlight, neutral annotation.
   - Avoid creating many unrelated colors page by page.

6. Chapter rhythm
   - Keep section labels and page rhythm stable so the audience can follow the talk.
   - Use repeated structure for orientation, but vary the content layout according to story phase.

7. Technical visual priority
   - For systems, networking, algorithms, and experiments, prefer architecture diagrams, workflows, timelines, comparison panels, and result plots over paragraphs.
   - A good slide should still make sense when the audience looks at it for 5 seconds.

## Generator Order

For every slide, generate in this order:

1. Story Phase
2. One Message
3. Audience 5-second takeaway
4. Visual type and visual area
5. Minimal supporting text
6. Speaker notes

Never start from a paper paragraph and then search for space on the slide.
